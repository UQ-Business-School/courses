Cell Referencing Week 2

Every Excel formula works with cell references. Understanding the three types of referencing is fundamental to writing formulas that copy correctly.

Relative References

A relative reference (e.g., A1) automatically adjusts when a formula is copied to another cell. The reference shifts by the same number of rows and columns as the formula was moved.

Example – Sales and Profits spreadsheet

If cell C2 contains =B2-A2 (Profits = Sales – Overhead) and you copy it to C3, it becomes =B3-A3 automatically.

  =B2-A2   ← typed in C2 (Profits Jan)
  =B3-A3   ← Excel auto-adjusts when copied to C3 (Profits Feb)

Absolute References

An absolute reference (e.g., $A$1) does NOT change when copied. Dollar signs $ lock both the column and the row.

Example – Tax Rate

In the Employee dataset, a single Tax Rate is stored in cell L1 (value: 0.0241). To calculate tax for each employee without the reference drifting:

=H2*$L$1

When this is copied down column I, H2 changes to H3, H4 etc., but $L$1 always points to the tax rate cell.

Mixed References

A mixed reference locks either the column ($A1) or the row (A$1) but not both.

  • $A1 — column A is locked; row changes when copied down
  • A$1 — row 1 is locked; column changes when copied right

Example – Multiplication Table (Mixed References)

To build a multiplication table where row headers are in column A and column headers are in row 1:

=$A2*B$1

This locks column A (so the row factor is always from column A) and row 1 (so the column factor is always from row 1). Copy this formula anywhere in the table and it works correctly.

Keyboard Shortcut

Press F4 while the cursor is inside a cell reference in the formula bar to cycle through all four reference types: A1 → $A$1 → A$1 → $A1 → A1.

⚠ Common Mistake Forgetting to use $ signs when a formula should always refer to one fixed cell (e.g., a tax rate, commission rate, or lookup table). This causes the #REF! or wrong-answer errors when the formula is copied down.

Basic Formulas & Functions Week 3

Arithmetic Operators

OperatorMeaningExample
+Addition=A1+B1
-Subtraction=A1-B1
*Multiplication=A1*B1
/Division=A1/B1
^Exponentiation (Power)=A1^2

Operator Precedence (BODMAS/PEMDAS)

Excel follows standard mathematical order: Brackets → Exponents → Division/Multiplication → Addition/Subtraction. Use parentheses () to override order.

=2+3*4     → result: 14   (not 20, because * happens before +)
=(2+3)*4   → result: 20

SUM

=SUM(number1, [number2], …)

Adds all numbers in a range. The most used function in Excel.

=SUM(B2:G2)       ← Sum of Jan–Jun Sales
=SUM(B2:B7)       ← Sum of a column
=SUM(B2:G7)       ← Sum of an entire block

Example – Sales & Profits (Weeks 2–3 dataset)

The course uses a dataset with Sales, Overhead/Expenses and Profits across months Jan–Jun.

  Sales row: Jan=120, Feb=180, Mar=260, Apr=240, May=300, Jun=500
  =SUM(B2:G2)  →  1600  (Total Sales)

AVERAGE

=AVERAGE(number1, [number2], …)

Returns the arithmetic mean of a range. Ignores blank cells but counts cells containing 0.

=AVERAGE(B2:G2)   ← Average monthly sales: 1600/6 = 266.67

YTD (Year-to-Date) Running Totals

A common exercise is building a cumulative (YTD) column. This requires an expanding absolute reference:

Cell B5 (YTD for Jan):  =B4          ← just Jan Profits
Cell C5 (YTD for Feb):  =$B$4+C4    ← or =SUM($B4:C4) using mixed ref

The standard pattern taught in this course uses a mixed absolute reference to lock the start of the range:

=SUM($B4:B4)   ← in Feb column becomes =SUM($B4:C4), etc.

Percentage Change

Formula for % change between two consecutive periods:

=(New - Old) / Old    or equivalently    =New/Old - 1

Example – % Sales Change

  Feb % Change:  =(C2-B2)/B2  →  (180-120)/120 = 0.5 = 50%
  Mar % Change:  =(D2-C2)/C2  →  (260-180)/180 ≈ 44.4%

Ratios

=B2/B3    ← Sales:Expenses ratio
⚠ Common Mistake: Dividing by zero If the denominator cell is blank or zero, Excel shows #DIV/0!. Wrap with IFERROR: 
=IFERROR(B2/B3, "N/A")

Ctrl+H opens Find & Replace. Useful for bulk text substitution across a sheet or workbook.

Outlining / Grouping Rows and Columns Supplementary

Grouping lets you collapse and expand sections of a spreadsheet, useful for hiding detail rows in a budget or report without deleting them.

  • Group: Select rows or columns → Data → Group
  • Ungroup: Data → Ungroup
  • Collapse/Expand: Click the – / + buttons on the left margin (for rows) or top margin (for columns)
  • Level buttons: Numbers (1, 2, 3…) in the top-left corner jump to different outline levels
  • Auto Outline: Data → Group → Auto Outline — Excel detects summary rows/columns automatically

Example – 2017 Budget Projections

Monthly columns (Jan, Feb, Mar…) are grouped under quarterly summary columns (1st Q, 2nd Q…). Collapsing level 1 hides the individual months and shows only quarterly totals.

3D References (Cross-Sheet)

A 3D reference lets a formula span multiple sheets. Syntax:

=SUM(Sheet1:Sheet4!B5)

This sums cell B5 across Sheet1, Sheet2, Sheet3, and Sheet4 — as used in the RegionalTotals sheet that sums East, Midwest, South, Pacific sheets.

Example – Regional Furniture Sales (Hard Task 1)

The workbook has four regional sheets (East, Midwest, South, Pacific) and a summary sheet. A 3D SUM formula on the summary sheet consolidates totals:

=SUM(East:Pacific!B3)

Excel Tables, Sorting & Filtering Week 3

Excel Tables

Converting a range to a Table (Insert → Table or Ctrl+T) provides automatic headers, banded rows, built-in filter dropdowns, and structured references.

Advantages of Tables

  • Formulas in table columns auto-fill to new rows
  • Structured references like [@Compensation] instead of raw cell addresses
  • Easy to sort and filter
  • Named reference: if the table is named "EmployeeTable", formulas can use =AVERAGE(EmployeeTable[Compensation])

Sorting

Data → Sort. You can sort by one or multiple columns. Always check whether you want to sort the whole table or just one column (sorting one column alone can misalign data!).

Filtering

Data → Filter adds dropdowns to column headers. Use these to show only rows meeting certain criteria (e.g., only "Full Time" employees in the Employee dataset).

Remove Duplicates

A built-in Data tab tool that identifies and removes rows with duplicate values in selected columns. Used in the fraud detection exercise (Week 4) to clean transaction data before analysis.

  • How to use: Select your data range → Data tab → Remove Duplicates
  • In the dialog, choose which columns to check — tick only "Transaction ID" to find duplicate transaction entries
  • Excel shows a message confirming how many duplicates were removed and how many unique values remain
  • Remove Duplicates permanently deletes rows — always work on a copy or undo (Ctrl+Z) if needed
💡 Why it matters Duplicate transaction IDs can indicate a system error, a manual data entry mistake, or fraudulent double-entry. Cleaning duplicates before running formulas ensures accurate counts and totals.

SORT Function

=SORT(array, [sort_index], [sort_order], [by_col])

Returns a sorted version of a range or array as a dynamic spill. Unlike Data → Sort, it does not modify the original data — the sorted result appears in a new location and updates automatically.

ArgumentDescription
arrayThe range or array to sort
sort_indexColumn (or row) number to sort by. Default = 1
sort_order1 = ascending (default), -1 = descending
by_colFALSE = sort by rows (default), TRUE = sort by columns

Example – Sort Students by Test Score (Descending)

=SORT(A2:B7, 2, -1)

Where A2:B7 contains Student and Test Score columns. Sort by column 2 (score), descending. Result spills automatically into adjacent cells.

💡 SORT vs Data → Sort
  • =SORT() is a formula — the original data stays unchanged and the sorted output is dynamic
  • Data → Sort physically rearranges the rows in place
  • =SORT() requires Excel 365 or Excel 2021+

Pie Charts

Used to visualise proportional data — for example, how total revenue is distributed across five store locations. Pie charts are created from a selected range of labels and values.

How to create a Pie Chart (e.g., Revenue by Store Location):

  • Select the labels and values (e.g., M25:N29 — store names and their revenue totals)
  • Go to the Insert tab → Chart → choose 2D Pie
  • Add a chart title (e.g., "Revenue by Store Location") by clicking on the default title
  • Move or resize the chart by dragging it
💡 When to use a Pie Chart
  • Use when showing how parts contribute to a whole (e.g., each store's share of total revenue)
  • Works best with 5 or fewer categories — too many slices makes it hard to read
  • Avoid when precise comparison between values matters — use a bar chart instead

Auto-Fill and Flash Fill

  • Auto-fill: Drag the fill handle (bottom-right corner of a cell) to copy a formula or continue a series.
  • Flash Fill (Ctrl+E): Excel detects a pattern and fills remaining cells (e.g., extracting first names from "Last, First" format).

IF, Nested IF, AND, OR Week 4

IF Function

=IF(logical_test, value_if_true, value_if_false)

Returns one value if a condition is TRUE, and another if it is FALSE.

ArgumentDescription
logical_testAny expression that evaluates to TRUE or FALSE. Uses comparison operators: =, >, >=, <, <=, <>
value_if_trueValue returned when condition is TRUE. Can be a number, text (in quotes), or another formula.
value_if_falseValue returned when condition is FALSE. Can be omitted (returns FALSE) but best practice is to always include it.

Example 1 – Simple IF: Shipping Charge (Home Furniture Sales)

Business rule: "If Subtotal > $2,000, shipping is free; otherwise, charge $25 shipping."

=IF(F3>2000, 0, 25)

Where F3 is the Subtotal column. If the order exceeds $2,000 the formula returns 0 (free shipping); otherwise $25.

Example 2 – Simple IF: Employee Bonus

Business rule: "Full Time employees with Job Rating 1 get a 5% bonus; otherwise no bonus."

=IF(AND(D2="Full Time", E2=1), H2*0.05, 0)

Where D = Status, E = Job Rating, H = Compensation.

Comparison Operators

OperatorMeaningExample
=Equal to=IF(A1=10,...)
>Greater than=IF(A1>100,...)
>=Greater than or equal to=IF(A1>=50,...)
<Less than=IF(A1<0,...)
<=Less than or equal to=IF(A1<=10,...)
<>Not equal to=IF(A1<>"Yes",...)

Nested IF

A nested IF places another IF as the value_if_false argument. Used when there are more than two possible outcomes.

=IF(test1, result1, IF(test2, result2, IF(test3, result3, default)))

Example – Job Rating Bonus (NestedIF sheet)

Business rules for bonus based on Job Rating (1=highest, 5=lowest):

  • Rating 1 → 10% bonus
  • Rating 2 → 7% bonus
  • Rating 3 → 5% bonus
  • Rating 4 → 2% bonus
  • Rating 5 → no bonus
=IF(B2=1, H2*0.10, IF(B2=2, H2*0.07, IF(B2=3, H2*0.05, IF(B2=4, H2*0.02, 0))))
⚠ Nesting Limit Excel allows up to 64 nested IFs, but deeply nested IFs are hard to read. For many conditions, consider VLOOKUP (approximate match) or IFS function instead.

AND Function

=AND(logical1, [logical2], …)

Returns TRUE only if ALL arguments are TRUE. Commonly nested inside IF.

=AND(D2="Full Time", E2=1)   ← TRUE only if Full Time AND rating is 1
=IF(AND(D2="Full Time", E2<=2), H2*0.05, 0)

OR Function

=OR(logical1, [logical2], …)

Returns TRUE if ANY argument is TRUE.

=OR(D2="Full Time", D2="Contract")  ← TRUE if Full Time or Contract
=IF(OR(E2=1, E2=2), H2*0.05, 0)    ← Bonus for ratings 1 or 2

Example – Compound IF: Bonus for Full Time with Years ≥ 10

Business rule: "Full Time employees with 10+ years AND Job Rating 1 or 2 get 8% bonus."

=IF(AND(D2="Full Time", C2>=10, OR(E2=1, E2=2)), H2*0.08, 0)

IFS Function (Modern Alternative to Nested IF)

=IFS(condition1, result1, condition2, result2, …, TRUE, default)

Tests multiple conditions in sequence. The last pair TRUE, default acts as the "else" case.

=IFS(B2=1, H2*0.10, B2=2, H2*0.07, B2=3, H2*0.05, B2=4, H2*0.02, TRUE, 0)

IFERROR

=IFERROR(value, value_if_error)

Returns a custom result if a formula produces an error; otherwise returns the normal result. Useful for hiding #N/A, #DIV/0!, #REF! etc.

=IFERROR(VLOOKUP(A2, $D$2:$E$10, 2, FALSE), "Not Found")

LET

=LET(name1, value1, [name2, value2, …], calculation)

Assigns names to values or formula results, then uses those names in a calculation. Makes complex formulas shorter, easier to read, and faster — the named value is calculated once rather than repeatedly.

ArgumentDescription
name1A name you define (like a variable) — must not be a cell reference or reserved word
value1The value or formula assigned to name1
calculationThe final formula that uses the named values — always the last argument

Example — Classifying a transaction without repeating the formula

Without LET (total calculated twice):
=IF(D7*E7>5000, "Review", "OK")

With LET (total calculated once, stored as "total"):
=LET(total, D7*E7, IF(total>5000, "Review", "OK"))

More complex example — rate gap classification:
=LET(
  target, XLOOKUP(D7, PathwayTitles, TargetRates, "No Pathway"),
  gap,    IF(target="No Pathway", "N/A", ROUND((E7-target)/target*100,1)),
  IF(gap="N/A", "Review", IF(gap < -10, "Promote", "On Track"))
)
💡 When to use LET
  • When the same sub-formula appears more than once in a formula — define it once with LET
  • When a formula is hard to read — naming intermediate values makes logic clearer
  • LET requires Excel 365 or Excel 2021+
  • The last argument is always the output — everything before it is just definitions

VLOOKUP Week 7

VLOOKUP ("Vertical Lookup") searches for a value in the first column of a table and returns a value from a specified column in the same row.

=VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
ArgumentDescription
lookup_valueThe value you are searching for (can be a cell reference, text, or number).
table_arrayThe range containing the lookup table. The value to search for MUST be in the FIRST column of this range. Almost always use absolute references ($A$2:$C$20).
col_index_numWhich column number (counting from 1 on the left of table_array) to return the result from.
range_lookupFALSE = Exact match (most common). TRUE or omitted = Approximate match (table must be sorted ascending).

Exact Match (FALSE)

Use when looking up a specific record by a unique key (product name, employee ID, etc.).

Example – Product Lookup (HOME 2015 Product Line)

Lookup table: columns = Product | Department | Category (starting in column K)

  =VLOOKUP("Bamboo Arm Chair", $K$2:$M$20, 2, FALSE)
  → Returns "Furniture"  (Department)

  =VLOOKUP(A2, $K$2:$M$20, 3, FALSE)
  → Returns Category for the product name in A2

Example – Building Email Addresses (Week 8 / Hard Task 5)

Consultant data: Consultant ID, Location, Email domain. Task: build each consultant's email.

  Consultant_details table ($G$3:$I$8):
    AWAN | Melbourne | salamander.com
    FJAM | Sydney    | endurance.com
    MMAO | Brisbane  | erebus.com
    JCHI | Brisbane  | proto.com
    BINA | Sydney    | fleet.com
    JCHA | Melbourne | tychos.com

  Consultant ID formula (from First Name + Last Name initials):
  =CONCAT(LEFT(B2,1), LEFT(C2,3))   → "AWAN" for Amos Wang

  Email formula:
  =CONCAT(B2, ".", C2, "@", VLOOKUP(CONCAT(LEFT(B2,1),LEFT(C2,3)), $G$3:$I$8, 3, FALSE))
  → "[email protected]"

Approximate Match (TRUE)

Use when the lookup value falls within a range (e.g., tax brackets, discount tiers, grade bands). The first column of the table MUST be sorted in ascending order.

Excel finds the largest value in the first column that is ≤ the lookup value.

Example – Discount/Reduction Rate (VLOOKUP-Approximate)

Reduction Rate table:

  Subtotal   | Reduction
  0          | 0%
  500        | 2%
  1000       | 3%
  1500       | 4.5%
  2000       | 6%
  2500       | 8%

Formula (Subtotal in D2, table in H:I):

=VLOOKUP(D2, $H$2:$I$7, 2, TRUE)

For a subtotal of $1,363.80 → looks up ≤ 1363.80 → finds 1000 → returns 3%.

Example – Grade Lookup (VLOOKUP-Introduction)

Grade cut-off table (must be ascending order):

  Mark Range | Grade
  0          | 1
  30         | 2
  47         | 3
  50         | 4
  65         | 5
  75         | 6
  85         | 7
  
=VLOOKUP(E12, $A$3:$B$9, 2, TRUE)

A student with 80 total marks → finds ≤ 80 → matches row 75 → Grade 6.

Example – Temperature Type of Day

  Temperature | Type of Day
  0           | Very Cold
  5           | Cold
  15          | Mild
  23          | Warm
  28          | Hot
  33          | Very Hot
  
=VLOOKUP(B2, $E$3:$F$8, 2, TRUE)

Temperature of 29°C → largest value ≤ 29 is 28 → returns "Hot".

⚠ VLOOKUP Limitations
  • Can only look up values in the first column of the table_array — cannot return values to the left.
  • With approximate match, if the table is NOT sorted ascending, results will be wrong (no error message!).
  • Fragile to column insertions: col_index_num is a hard-coded number that breaks if columns are added/removed.
  • For looking to the left or more flexibility, use XLOOKUP instead.

Common VLOOKUP Errors

ErrorCauseFix
#N/ALookup value not found in first columnCheck spelling, data type (text vs number), or use IFERROR
#REF!col_index_num exceeds number of columns in table_arrayReduce col_index_num or expand table_array
Wrong valueForgot FALSE for exact match (TRUE is default)Add FALSE as 4th argument
Wrong valueTable not locked with $ signs — shifts when copiedUse absolute references for table_array

XLOOKUP Week 8

XLOOKUP is a modern, more powerful replacement for VLOOKUP. It can look left, look up or across, return multiple columns, and has a built-in "not found" argument.

=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])
ArgumentDescription
lookup_valueThe value to search for.
lookup_arrayThe single column (or row) to search in — does NOT need to be the first column.
return_arrayThe single column (or row) to return the result from.
if_not_foundOptional. Text or value to return if no match found (replaces IFERROR wrapper).
match_mode0 = Exact match (default), -1 = Exact or next smaller, 1 = Exact or next larger, 2 = Wildcard.
search_mode1 = First to last (default), -1 = Last to first, 2 = Binary ascending, -2 = Binary descending.

Example – Student University Dataset (XLOOKUP exercises)

Dataset columns: Student ID | Name | Birth Year | Email | Course code | Course name | School

  Q: In which school is James studying?
  =XLOOKUP("James", B4:B20, G4:G20)
  → Returns "Law department"

  Q: What is the Student ID for Hanna?
  =XLOOKUP("Hanna", B4:B20, A4:A20)
  → Returns "S04"

  Q: Can we retrieve the student ID for the person studying in "Information system department"?
  =XLOOKUP("Information system department", G4:G20, A4:A20)
  → Returns "S07"  (XLOOKUP can look left — VLOOKUP cannot!)

  Q: With not-found message:
  =XLOOKUP("Unknown", B4:B20, G4:G20, "Student not found")
  → Returns "Student not found"

Example – Staff Directory (XLOOKUP answer file)

Dataset: Staff ID | First name | Last name | Office location | Country | Department | Position title

  =XLOOKUP(129300, A2:A7, C2:C7)
  → Returns "Fernandes"  (Last name for Staff ID 129300)

Example – Football Goals (Part 2 / Hard Task 3 – XLOOKUP variant)

Data is stored in a "Name|Competition" format: e.g., "Cavani|League", "Cavani|World Cup"

  =XLOOKUP(D2&"|League", A:A, B:B, 0)
  → Finds "Cavani|League" and returns 120

  =XLOOKUP(D2&"|World Cup", A:A, B:B, 0)
  → Finds "Cavani|World Cup" and returns 14

  Total Goals =
  =XLOOKUP(D2&"|League",A:A,B:B,0) + XLOOKUP(D2&"|World Cup",A:A,B:B,0)
  → For Cavani: 120 + 14 = 134

For "Suarez" who has "Error in data" for World Cup, wrapping with IFERROR returns 0 or a custom message.

COUNTIF, COUNTIFS & COUNTA Week 5

COUNTIF

=COUNTIF(range, criteria)

Counts the number of cells in a range that meet a single condition.

Example – Home Products Dataset (COUNTIF sheet)

Dataset: Category | Department | Product | Size | Color | Price | Sold | Revenue

  How many items are in the Kitchen department?
  =COUNTIF(B:B, "Kitchen")

  How many products have revenue over $300,000?
  =COUNTIF(H:H, ">300000")

  How many products are in the Furniture category?
  =COUNTIF(A:A, "Furniture")

Criteria Options for COUNTIF

CriteriaMeaningExample
"Kitchen"Exact text match=COUNTIF(B:B,"Kitchen")
">300000"Greater than 300,000=COUNTIF(H:H,">300000")
"<>Full Time"Not equal to=COUNTIF(D:D,"<>Full Time")
"F*"Starts with F (wildcard)=COUNTIF(A:A,"F*")
"*chair*"Contains "chair"=COUNTIF(C:C,"*chair*")
A reference cellMatch value in a cell=COUNTIF(B:B,A1)
">"&A1Greater than cell value=COUNTIF(H:H,">"&D1)

COUNTIFS

=COUNTIFS(range1, criteria1, [range2, criteria2], …)

Counts rows where ALL conditions are met simultaneously (AND logic).

How many Full Time employees with Job Rating 1?
=COUNTIFS(D:D, "Full Time", E:E, 1)

How many Kitchen items with revenue > $50,000?
=COUNTIFS(B:B, "Kitchen", H:H, ">50000")

COUNT

=COUNT(value1, [value2], …)

Counts cells that contain numbers only. Ignores text, blanks, and errors. Use when you need to know how many numeric entries exist in a range.

Example – Count Numeric Salary Entries

=COUNT(H2:H100)   →  returns the number of cells in column H that contain numbers

Useful for checking whether all employees have a salary entered (text or blank entries are not counted).

💡 COUNT vs COUNTA vs COUNTIF
  • COUNT — counts cells containing numbers only
  • COUNTA — counts all non-empty cells (any data type)
  • COUNTIF — counts cells meeting a specific condition
  • COUNTIFS — counts cells meeting multiple conditions

COUNTA

=COUNTA(value1, [value2], …)

Counts all non-empty cells (regardless of data type — text, numbers, dates, errors all count). Useful when you want to count filled cells, not just numbers.

Example – Student Registration Dataset (COUNTA sheet)

Columns: Student ID | Student Name | Registration Status | Major | Contact Email | Notes

  Count students with a Registration Status (non-empty):
  =COUNTA(C2:C9)  →  6

  Count non-empty cells in the row for student ST005:
  =COUNTA(A6:F6)  →  4

  Count students who have provided their major:
  =COUNTA(D2:D9)  →  5

  Total non-empty cells in the entire dataset:
  =COUNTA(A2:F9)  →  42
💡 COUNT vs COUNTA vs COUNTIF
  • COUNT — counts cells containing numbers only
  • COUNTA — counts all non-empty cells (any data type)
  • COUNTIF — counts cells meeting a specific condition
  • COUNTIFS — counts cells meeting multiple conditions

SUMIF & AVERAGEIF

=SUMIF(range, criteria, [sum_range])

CONCAT / CONCATENATE / & Operator

=CONCAT(text1, [text2], …) or =text1 & text2

Joins (concatenates) text strings together.

=CONCAT(B2, ".", C2, "@", "company.com")
="Amos" & "." & "Wang"   →  "Amos.Wang"

=LEFT(text, [num_chars])

Returns the specified number of characters from the start (left) of a text string.

=RIGHT(text, [num_chars])

Returns the specified number of characters from the end (right) of a text string.

=RIGHT("example.com", 3)  →  "com"

=MID(text, start_num, num_chars)

Returns a specific number of characters from the middle of a text string, starting at a given position.

Returns the number of characters in a text string (including spaces).

Returns the position of a text string within another. Case-sensitive. Returns #VALUE! if not found.

Removes all leading, trailing, and extra internal spaces from text. Essential for cleaning data before lookups.

ROUND

=ROUND(number, num_digits)

Rounds a number to a specified number of decimal places. Uses standard rounding (0.5 rounds up).

=ROUND(123.456, 2)   →  123.46
=ROUND(67.23456, 2)  →  67.23
=ROUND(92.0049, 0)   →  92     ← round to whole number
=ROUND(92.5, 0)      →  93

ROUNDDOWN

=ROUNDDOWN(number, num_digits)

Always rounds towards zero, regardless of the decimal value.

=ROUNDDOWN(12345.67, 0)  →  12345  (always truncates)
=ROUNDDOWN(8765.99, 0)   →  8765

ROUNDUP

=ROUNDUP(number, num_digits)

Always rounds away from zero.

=ROUNDUP(12345.01, 0)  →  12346

MOD Supplementary

=MOD(number, divisor)

Returns the remainder after dividing number by divisor. Useful for alternating patterns (odd/even, day cycles).

=MOD(10, 3)   →  1   (10 = 3×3 + 1)
=MOD(6, 5)    →  1   → Monday (session 6 cycles back to day 1)

Mapping session numbers to weekdays (Mon=1...Fri=5):
=MOD(A2-1, 5)+1  →  gives 1–5 cycling

SUMPRODUCT Supplementary

=SUMPRODUCT(array1, [array2], …)

Multiplies corresponding elements of arrays and returns the sum of those products. Perfect for weighted calculations and totals.

Example – Inventory Revenue

  Items:      Books(100), Pen set(200), Pencil colors(56), Pens(120), Notepads(54), Rubbers(600)
  Price/unit: 10, 15, 25, 9, 26, 3

  =SUMPRODUCT(B2:B7, C2:C7)

  = 1000 + 3000 + 1400 + 1080 + 1404 + 1800

PMT Supplementary

=PMT(rate, nper, pv, [fv], [type])

Calculates the fixed periodic payment for a loan or investment, given a constant interest rate. Used in the Home Loan exercise to calculate monthly repayments.

ArgumentDescription
rateInterest rate per period (annual rate ÷ 12 for monthly payments)
nperTotal number of payment periods (years × 12 for monthly)
pvPresent value — the loan amount (enter as negative for a loan)
fvFuture value after last payment. Default = 0 (loan fully repaid)
type0 = payment at end of period (default); 1 = start of period

Example – Home Loan Monthly Repayment

  Loan: $20,000 | Annual rate: 5% | Term: 5 years (60 months)
  Monthly rate = 5% / 12 = 0.4167%

  Or using cell references (e.g., rate in B2, term in B3, amount in B1):

Named Ranges Week 7

A Named Range assigns a meaningful name to a cell or range, making formulas more readable and easier to maintain.

Creating Named Ranges

  • Select the cell/range → click the Name Box (left of formula bar) → type name → Enter
  • Or: Formulas → Name Manager → New
  • Or: Formulas → Define Name

Rules for Names

  • Must start with a letter or underscore (_)
  • No spaces (use underscore: Tax_Rate)
  • Cannot look like a cell reference (e.g., "A1" is not allowed)
  • Case-insensitive: TaxRate = taxrate

Using Named Ranges in Formulas

Instead of: =H2*$L$1
Use:        =H2*TaxRate

Instead of: =VLOOKUP(A2, $D$2:$F$20, 3, FALSE)
Use:        =VLOOKUP(A2, ProductTable, 3, FALSE)
💡 Benefit: Named ranges are always absolute by default. When you use a named range in a formula, it automatically refers to the same cells wherever the formula is used.

Top Misconceptions

1. Forgetting $ signs (Absolute References)

The most common beginner mistake. When a formula must always reference the same cell (tax rate, lookup table, etc.) and you copy it without locking with $, the reference drifts.

Fix: Use F4 to add $ signs. For lookup tables, always use $A$2:$C$20 syntax.

2. VLOOKUP Default Range_Lookup = TRUE

If you omit the 4th argument, VLOOKUP defaults to TRUE (approximate match). This gives wrong answers for exact-match lookups, often without any error message.

Fix: Always explicitly type FALSE for exact match: =VLOOKUP(A2, $D$2:$F$10, 2, FALSE)

3. VLOOKUP Approximate Match with Unsorted Data

Approximate match (TRUE) requires the first column to be sorted ascending. If it is not sorted, VLOOKUP returns incorrect results silently.

4. Text vs Number Mismatch

A lookup value stored as text (e.g., "123") will not match the same value stored as a number (123). Common when IDs are imported from other systems.

Fix: Use VALUE() to convert text to number, or TEXT() to convert number to text. Check alignment: numbers align right by default, text aligns left.

5. COUNTIF Text Criteria Must Be in Quotes

Criteria like "Full Time" must be in double quotes. Numeric criteria can be in quotes when using operators: ">100".

WRONG:  =COUNTIF(D:D, Full Time)     ← #NAME? error
RIGHT:  =COUNTIF(D:D, "Full Time")

6. Nested IF Order Matters

Excel tests conditions in sequence — once a condition is TRUE, it stops. Put the most restrictive/specific condition first.

WRONG (overlapping):
=IF(A1>=50, "Good", IF(A1>=75, "Excellent", "Poor"))
← A1=80 returns "Good" because ≥50 is tested first!

RIGHT:
=IF(A1>=75, "Excellent", IF(A1>=50, "Good", "Poor"))

7. SUM vs SUMIF vs SUMPRODUCT

  • SUM — adds everything in a range, no conditions
  • SUMIF — adds only matching rows based on one condition
  • SUMIFS — adds matching rows based on multiple conditions
  • SUMPRODUCT — multiplies arrays element-by-element, then sums

8. Circular References

A formula that refers to its own cell (directly or indirectly) creates a circular reference. Excel usually warns about this. They must be resolved — either the formula logic is wrong, or iterative calculation must be enabled intentionally.

9. Goal Seek Cannot Change a Cell Containing a Formula

The "By changing cell" must be a simple input cell (no formula). If it contains a formula, Goal Seek cannot work.

10. XLOOKUP Not Available in Older Excel

XLOOKUP requires Excel 365 or Excel 2019+. In older versions (2016, 2013), use VLOOKUP + IFERROR instead. The course exercises may note which version to use.

11. AVERAGE Ignores Blanks; SUMPRODUCT Does Not

AVERAGE skips blank cells. But if you try to use SUMPRODUCT and divide by COUNT, make sure to handle blanks correctly.

12. Formatting ≠ Value Change

Applying a percentage format to a cell containing 0.25 displays "25%". But applying percentage format to a cell already showing "25" displays "2500%". The underlying value and the format are separate.
