Part A: Information Systems Theory

A1: The Systems Development Life Cycle (SDLC)

The Systems Development Life Cycle (SDLC) is a structured framework for planning, creating, testing, and deploying information systems. The Waterfall methodology is one approach to implementing the SDLC.

Key Characteristics of Waterfall

  • Highly sequenced: Phases flow downward like a waterfall
  • Sequential: Each phase must be completed before the next begins
  • Gate-controlled: Project control through approval stages or "gates"
  • Documentation-heavy: Comprehensive documentation at each phase
  • Change-resistant: Changes become more expensive the later they occur

The 5 Phases of Waterfall

Phase 1: Systems Definition

  • Senior management defines broad goals and scope
  • Assess feasibility (technical, economic, operational)
  • Form a project team
  • Create initial project plan
  • Deliverable: Project plan with defined scope
  • Gate: Senior management approval

Phase 2: Requirements Analysis

  • Gather detailed requirements from users and stakeholders
  • Document all user needs in a detailed report
  • Critical phase: Incorrect requirements lead to flawed systems
  • Deliverable: Approved user requirements document
  • Gate: Stakeholder sign-off

Phase 3: Design, Development, and Testing

  • Convert requirements into design specifications
  • Build/develop the system
  • Conduct rigorous testing
  • Very complex phase
  • Deliverable: System design documentation, tested software/hardware

Phase 4: Implementation

  • Deploy the system for use
  • Train users
  • System goes "live"
  • Deliverable: Operational system

Phase 5: Maintenance

  • All work after implementation
  • Bug fixes, updates, support
  • Often the MOST EXPENSIVE phase
  • Often neglected in planning
  • May initiate a new SDLC cycle
Key Concept: Cost of Changes Over Time
The later errors or changes are discovered in the SDLC, the more expensive they become to fix. A requirements error caught in maintenance costs far more than one caught during requirements analysis.

Strengths of Waterfall

  • Clear structure and milestones
  • Comprehensive documentation
  • Works well for projects with stable, well-understood requirements
  • Easier to manage and track progress

Limitations of Waterfall

  • Inflexible to changing requirements
  • Users don't see the product until late in the process
  • Long time between project start and delivery
  • High risk if requirements are wrong

A2: Agile and Scrum Methodology

Agile is an iterative approach to software development that emphasises flexibility, collaboration, and rapid delivery. Scrum is a specific framework within Agile.

Key Characteristics of Scrum

  • Sprints: Short development cycles (typically 2-4 weeks)
  • Cross-functional teams: Teams work together collaboratively
  • Frequent delivery: Working software delivered regularly
  • Continuous feedback: From customers/stakeholders throughout
  • Adaptable: Can respond to changing requirements
Key Concept: Minimum Viable Product (MVP)
The goal is to deliver a potentially shippable product at the end of each sprint that can be shown to the customer for feedback. This feedback influences the next sprint.

Benefits of Scrum over Waterfall

  • Higher success rate for IS projects
  • Faster delivery (no months-long wait)
  • More flexibility to adapt to changes
  • Customer sees working product early and often
  • Continuous improvement based on feedback

Potential Downsides of Scrum

  • Often not implemented correctly
  • Mixing Waterfall and Scrum approaches doesn't work well
  • "Product owners" may lack authority to drive business decisions
  • Resistance from employees who prefer individual recognition over team collaboration
  • Potential for burnout due to intensity of sprints
  • Less comprehensive documentation

Part B: Excel Functions

B1: VLOOKUP Function

Syntax

=VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])

Arguments Explained

1. lookup_value (required)

  • The value to search for in the FIRST column of the table
  • Can be a value, cell reference, or text
  • If lookup_value is smaller than the smallest value in the first column, returns #N/A error

2. table_array (required)

  • The range of cells containing the data
  • Can be a range reference (e.g., A2:D8) or a named range
  • The FIRST column of this range is always searched
  • Should use absolute referencing ($) when copying formulas

3. col_index_num (required)

  • The column number in the table from which to return a value
  • 1 = first column, 2 = second column, etc.
  • If less than 1: returns #VALUE! error
  • If greater than number of columns: returns #REF! error

4. range_lookup (optional but important)

ValueMatch TypeData Requirement
TRUE or 1 or omittedApproximate matchData MUST be sorted ascending
FALSE or 0Exact matchData does not need to be sorted
Critical Points:
  • VLOOKUP can only look RIGHT (returns values from columns to the right of the lookup column)
  • First column of table_array is ALWAYS the lookup column
  • TRUE vs FALSE fundamentally changes how the function works
  • Approximate match (TRUE) requires sorted data in ascending order

Example Formulas from Course

Grade lookup (approximate match):
=VLOOKUP(E13,$A$3:$B$9,2,TRUE)
Looks up the mark in E13, finds the grade from a grade boundary table.
Temperature description (approximate match):
=VLOOKUP(A8,$D$7:$F$12,3,TRUE)
Looks up temperature, returns description (e.g., "Hot", "Cold").
Type of day lookup (exact match):
=VLOOKUP(A18,$D$18:$E$23,2,FALSE)
Looks up exact day type, returns corresponding value.

B2: Combining IF with VLOOKUP

Why combine them? VLOOKUP returns #N/A when the lookup_value is outside the table range. Wrapping VLOOKUP in an IF function handles these edge cases.

Pattern

=IF(condition, value_if_true, VLOOKUP(...))

Example from Course

=IF(A13<0,"Below zero",VLOOKUP(A13,$D$7:$F$12,3,TRUE))

This formula:
  1. First checks if temperature is below zero
  2. If yes: returns "Below zero"
  3. If no: performs the VLOOKUP
Key Principle: This technique ensures formulas work for ALL possible input values, not just those within the lookup table range.

B3: XLOOKUP Function

Syntax

=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])

Arguments Explained

1. lookup_value (required) - The value to search for

2. lookup_array (required) - The range to search (single column or row)

3. return_array (required) - The range from which to return values. Can be multiple columns.

4. if_not_found (optional) - Text to display if no match found. Avoids #N/A errors.

5. match_mode (optional)

ValueMeaning
0Exact match (default)
-1Exact match or next smaller
1Exact match or next larger
2Wildcard match (* and ?)

6. search_mode (optional)

ValueMeaning
1Search first to last (default)
-1Search last to first (bottom to top)
2Binary search ascending
-2Binary search descending

Advantages of XLOOKUP over VLOOKUP

  • Can look LEFT or RIGHT (not limited to looking right)
  • Separate lookup_array and return_array (more flexible)
  • Built-in error handling with if_not_found
  • Supports wildcards for partial matches
  • Can search from bottom to top
  • Can return multiple columns at once

B4: Named Ranges

What are named ranges? Named ranges assign a meaningful name to a cell or range of cells, making formulas easier to read and maintain.

How to Create

  1. Go to Formulas tab
  2. Click Name Manager
  3. Click New
  4. Enter name and select range
  5. Click OK

Naming Rules

  • No spaces allowed (use underscores: Temperature_Range_Table)
  • Must follow Excel naming conventions
  • Cannot start with a number
  • Cannot use cell references as names

Benefits

  • More readable formulas: =VLOOKUP(A1,Temperature_Range_Table,2,TRUE)
  • Easier to maintain than absolute references
  • Self-documenting
  • Professional practice
Example named ranges from course:
Temperature_Range_Table
Type_of_Day_Table

B5: Cell Referencing

Types of References

TypeExampleBehaviour When Copied
RelativeA1Changes based on new position
Absolute$A$1Does NOT change
Mixed (column locked)$A1Column stays, row changes
Mixed (row locked)A$1Column changes, row stays

How References Change When Copied

If you copy a formula one column right and one row down:

OriginalAfter Copy
A1B2
$A$1$A$1
$A1$A2
A$1B$1

Why This Matters for VLOOKUP

The table_array argument should use absolute references so that when the formula is copied down, the table reference stays the same.

Example: =VLOOKUP(A8,$D$7:$F$12,3,TRUE)
  • A8 is relative (changes to A9, A10, etc. when copied down)
  • $D$7:$F$12 is absolute (stays the same when copied)

Mixed Reference Example

Formula: =$C5*(1-E$4)
  • $C5: Column C is locked, row is relative
  • E$4: Column is relative, row 4 is locked
  • This allows one formula to be copied across and down a grid to calculate different discount percentages for different prices

Part C: Common Errors and Troubleshooting

#N/A Error

Cause: VLOOKUP/XLOOKUP cannot find the lookup value

Solutions:

  • Check spelling of lookup value
  • For VLOOKUP with TRUE: ensure lookup_value isn't smaller than smallest table value
  • Use IF wrapper or XLOOKUP's if_not_found parameter

#REF! Error

Cause: col_index_num exceeds columns in table_array

Solution: Check that col_index_num doesn't exceed the number of columns

#VALUE! Error

Cause: col_index_num is less than 1, or wrong data type

Solution: Ensure col_index_num is at least 1

Wrong Results (No Error Shown)

Cause: Using approximate match (TRUE) with unsorted data

Solution: Sort data ascending OR use exact match (FALSE)

Formula Returns Same Value When Copied

Cause: Used relative reference where absolute was needed

Solution: Add $ signs to lock the table_array reference

Part D: Sample Quiz Questions

Information Systems Theory Questions

Multiple Choice

  1. Which phase of the Waterfall methodology is typically the most expensive?
    A) Requirements Analysis B) Implementation C) Maintenance D) Systems Definition
    Answer: C) Maintenance
  2. What does MVP stand for in Agile/Scrum?
    A) Most Valuable Process B) Minimum Viable Product C) Maximum Value Performance D) Managed Version Plan
    Answer: B) Minimum Viable Product
  3. Which methodology uses "sprints"?
    A) Waterfall B) SDLC C) Scrum D) Systems Definition
    Answer: C) Scrum

Short Answer

  1. Explain why errors discovered late in the SDLC cost more to fix than those discovered early.
    Answer: Later discovery means more work has been built on flawed foundations. Fixing requires reworking design, code, testing, and potentially retraining users — all adding cost.
  2. List the 5 phases of the Waterfall methodology in order.
    Answer: 1) Systems Definition, 2) Requirements Analysis, 3) Design/Development/Testing, 4) Implementation, 5) Maintenance
  3. Give two reasons why Scrum projects might have higher success rates than Waterfall projects.
    Answer: Frequent feedback allows early correction; customers see working product regularly; teams can adapt to changing requirements; problems identified early.

Scenario-Based

  1. A company is developing a new payroll system. The requirements are well-understood, unlikely to change, and senior management wants detailed documentation at each stage. Would you recommend Waterfall or Agile? Explain.
    Answer: Waterfall would be appropriate because requirements are stable and well-understood, documentation is valued, and the sequential approach suits projects with low uncertainty.
  2. A startup is building a new mobile app. They're not sure exactly what features users want and expect to make many changes based on user feedback. Which methodology would you recommend and why?
    Answer: Agile/Scrum because it allows for changing requirements, frequent delivery for user feedback, and iterative improvement based on real user responses.

Excel Function Questions

Formula Writing

  1. Write a VLOOKUP formula that looks up the value in cell A2 in the range D1:F10, returns the value from the third column, and finds an exact match.
    Answer: =VLOOKUP(A2,$D$1:$F$10,3,FALSE) or =VLOOKUP(A2,$D$1:$F$10,3,0)
  2. Write an XLOOKUP formula that looks up "Smith" in column A, returns the value from column C, and displays "Not found" if there's no match.
    Answer: =XLOOKUP("Smith",A:A,C:C,"Not found")
  3. Write a formula that uses IF and VLOOKUP to return "Invalid" if the value in A1 is less than 0, otherwise performs a VLOOKUP in the range D1:E10.
    Answer: =IF(A1<0,"Invalid",VLOOKUP(A1,$D$1:$E$10,2,TRUE))

Error Diagnosis

  1. A student's formula returns #N/A: =VLOOKUP("Marketing",A2:C10,2,TRUE). The value "Marketing" exists in column A. What might be causing the error?
    Answer: Using TRUE (approximate match) with text data that may not be sorted alphabetically. Should use FALSE for exact match with text lookups.
  2. A formula returns #REF!: =VLOOKUP(A1,D1:E10,3,FALSE). What's wrong?
    Answer: col_index_num is 3, but the table_array (D1:E10) only has 2 columns. Change to 2 or expand the range.

Fill-in-the-Blank

  1. In VLOOKUP, when the fourth argument is set to ______, the data in the first column must be sorted in ascending order.
    Answer: TRUE (or 1)
  2. To lock both the column and row when copying a formula, use ______ referencing.
    Answer: absolute (e.g., $A$1)
  3. XLOOKUP can search from bottom to top by setting the search_mode parameter to ______.
    Answer: -1

Conceptual

  1. What is the main advantage of using named ranges instead of absolute cell references?
    Answer: Named ranges make formulas more readable and self-documenting, easier to maintain, and less prone to errors when ranges change.
  2. Why should you use absolute references for the table_array in VLOOKUP when you plan to copy the formula?
    Answer: Without absolute references, the table_array range shifts when copied, causing the formula to reference wrong cells or return errors.
