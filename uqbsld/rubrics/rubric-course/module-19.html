1

Integration Strategy Selector

GA criteria can be integrated into rubrics via three approaches. For each scenario, decide which strategy best fits:

Dedicated Row: Add a separate criterion just for the GA (e.g., "GA2: Critical Analysis" as its own row). Use when there is no existing criterion that overlaps with the GA.

Embedded (Smushed): Weave GA language into an existing criterion that overlaps perfectly (mark with *). Use when a rubric criterion and GA sub-criterion assess the same competency.

Hybrid: Some GAs as dedicated rows, others embedded in existing criteria. Use when some GA outcomes overlap with existing criteria and others don't.

Decision Framework: How to Choose

Ask yourself three questions:

  • Does my rubric already measure this GA outcome? If yes → Embedded (smush it in, add the asterisk).
  • Does my rubric partially overlap? If yes → Hybrid (embed the overlapping part, add a dedicated row for the rest).
  • Is this GA outcome entirely separate from my existing criteria? If yes → Dedicated Row.

The goal is to avoid double-counting (testing the same thing twice) while ensuring the GA is genuinely assessed, not just bolted on.

The Smush in Action: Text Transform

Take the MGTS2501 "Critical Analysis" criterion and GA2 "Critical Analysis" sub-criterion—they overlap perfectly. Combine them into a single descriptor that covers both the assessment criterion AND the GA requirements. Add the asterisk (*).
