    Program at a glance

    The Master of Business Analytics is a program for professionals who want to turn data into business value. It is cross-disciplinary, bringing together business, data science and ethics/philosophy, and it uses real case studies plus an industry-based consulting project to connect analysis to real organisational decisions.

    Who it is for

    Professionals building analytics capability in business contexts, including leadership, technical application and the responsible use of data.

    What students develop

    Business analytics, data visualisation, machine-learning related capability, and the ability to communicate across business and technical domains.

    Why this matters for your course

    Courses should make the business use of analytics visible, not present content as isolated technical exercises.

    How the program is delivered

    MBusAn is designed as a 100% online, part-time program. Students study while balancing other commitments, and the delivery model includes live interactive elements, case-based learning, coding and programming support, and an industry-facing capstone experience.

    Online delivery

    The course shell carries much of the orientation, pacing, and signposting work.

    Part-time rhythm

    Clear sequencing matters for students balancing study with professional and personal commitments.

    Live sessions

    Students need obvious access to Zoom links, recordings, and any supporting session materials.

    Applied learning

    Case studies, practical tasks, and projects should be clearly framed as part of the learning journey.

    Support and resources

    Guides, staff details, readings, and recurring course tools should be easy to find throughout the semester.

    Helpful tipIf students can quickly understand what to do each week and where to find key resources, the course experience will feel more manageable and consistent.

    Project ecosystem

    Helpful tipIf students need to access it during learning, it should be in Ultra. If it is for the team to build or manage the course, it belongs in working spaces.

    Weekly teaching checklist

    A light-touch checklist you can use each week, especially in the Prepare folder and before the live session.

    • Review the Required Readings and any linked resources.
    • Review the module content to make sure instructions, links and dates still look right.
    • Respond to student questions in Ed Discussion.
    • Note any Ed Discussion questions that could work well as oral discussion in the live session.
    • Check Padlets (if used) and provide feedback to students.
    • Confirm live session links, slides and supporting materials are ready to go.
    • Check for any broken links, missing files or release issues in the module.
    • Flag any content, signposting or delivery issues that may need follow-up after teaching.
    Helpful tipA quick weekly check can help surface small issues early and keep the student experience consistent.

    Top-level / course site

    These items sit above the teaching modules and establish shared course access points.

    Course Coordinator Hub / Staff Only Area

    Hidden from students. Holds coordinator onboarding, guides and staff-facing notes.

    Course Profile

    The official course profile and course-level overview.

    Assessment

    Assessment Summary first, then one clearly labelled folder per task.

    • Description
    • Rubric
    • Related files

    Course Resources

    Shared materials students may need throughout the semester.

    • Guides
    • Live Sessions
    • Ed Discussion
    • Talis Course Reading List
    • Course Staff
    Use this area forCourse-wide tools, support materials, shared links, and administrative orientation.
    Do not use this area forWeekly teaching content, scattered readings, or loose activity pages that belong inside a module.

    Folder descriptions at top level

    • Assessment — students should find the assessment summary first, then one folder per task.
    • Course Resources — use for guides, live session links/recordings, Ed Discussion, the full Talis list and course staff information.

    Module level

    Modules should be organised consistently so students can predict where to find learning content every time.

    Module folderModule number + topic
    Module descriptionWeek(s) covered and key details
    Module IntroductionTopics, outcomes and important tasks
    Learning foldersPrepare and Attend / Live Session
    Readings folderModule-specific Talis/HBR access and reading directions
    What to include
    • Clear naming for searchability
    • Descriptions that orient students to the week(s) and expectations
    • A tidy top level with resources grouped into folders
    • A dedicated Readings folder where relevant
    What to avoid
    • Flat lists of files inside a module
    • No introduction document
    • Readings or live-session files scattered through pages

    Standard module order

    Module 3: Data VisualisationTop-level module folder named as module number + topic.
    Module IntroductionOrient students to topics, learning outcomes and important tasks.
    Prepare folderOnline learning materials and guided pre-class work.
    Attend / Live Session folderMaterials for tutorials or live sessions.
    Readings folderModule-specific Talis/HBR access and reading directions.

    This is the standard sequence to follow when building a module from scratch.

    Activity prompts and folder descriptions

    • Required reading via Talis: To view all course readings, navigate to the course reading list under Course Resources or in the Readings subfolder within each module.
    • HBR resource: Harvard Business Review articles can be found in the Readings subfolder within each module.
    • Ed Discussion: Access Ed Discussions via the link at the top of the course Blackboard page. Comment on the Module X post to share your response.

    Document level

    Use this level for text edits and content updates within documents and normal Ultra text items.

    Use this level for
    • Updating wording, instructions and examples
    • Refreshing links and reading references
    • Correcting errors or outdated information
    • Improving clarity, sequencing and consistency
    Keep in mind
    • Signposting changes should be discussed with a learning designer
    • If the custom HTML treatment needs to change, do not alter it in isolation
    • Keep substantial subject-matter content in standard Ultra formats where possible

    Bespoke MBusAn signposting

    • Social Activity — custom HTML signposting used to frame peer interaction or discussion.
    • Live Session — custom HTML signposting used to frame synchronous class activity.
    • Required Reading and Dive Deeper — custom HTML signposting used to distinguish essential vs optional reading.
    • Stop and Think — reflection prompt usually placed at the end of a document.
    • Looking Ahead — brief reflection or forward-looking prompt, often before a live session or at a module transition.
    Coordinator guidanceUpdate the text inside documents as needed, but discuss signposting changes with a learning designer.

    Image attributions

    • Keep attributions for all images, including free or openly licensed sources.
    • If you replace an image, update the attribution at the same time.
    • If an image carries meaning, include meaningful alt text.

    Bloom’s Taxonomy

    • Remember — define, list, identify.
    • Understand — explain, summarise, describe.
    • Apply — solve, demonstrate, use.
    • Analyse — compare, differentiate, analyse.
    • Evaluate — justify, critique, evaluate.
    • Create — design, develop, produce.
    Helpful tipMatch verbs to the cognitive demand you actually want, then make sure the assessment asks for that same level.

    Backward Design

    • Stage 1 — desired results.
    • Stage 2 — acceptable evidence.
    • Stage 3 — learning experience.
    Helpful tipStart with the learning outcomes, then check assessment, then review the supporting activities and materials.

    Images and attributions

    • Keep attributions for all images, including free or openly licensed sources.
    • If you replace an image, update the attribution at the same time.
    • If an image carries meaning, include meaningful alt text.
    • Avoid decorative images that add visual noise without helping students navigate or understand the material.

    Accessibility

    • Use real headings, not bold-only pseudo-headings.
    • Use descriptive link text.
    • Provide captions for video and audio where needed.
    • Make sure content works on a phone and by keyboard.
    • Keep contrast strong enough for readability.

    Copyright

    • Prefer library-licensed and openly licensed resources.
    • Link rather than upload where licences require it.
    • Use Talis Aspire early enough for the library workflow.
    • If unsure, ask the library copyright or learning resources contacts rather than guessing.
